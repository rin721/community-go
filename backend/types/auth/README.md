# types/auth 目录说明

`types/auth` 定义跨传输层、应用装配层和业务模块共享的认证上下文类型。

## 职责

- `Principal`：表示当前请求已认证的调用主体。
- `PermissionContext`：表示一次权限判断所需的产品、作用域、对象和动作。

## 使用规范

- 中间件、HTTP 路由装配和非 IAM 业务模块需要读取当前主体或发起权限判断时，应依赖本包类型。
- IAM 模块可以把这些类型作为自身服务接口的一部分，但认证、授权、会话、角色和权限领域规则仍留在 IAM 模块内。
- 不要在本包新增用户、角色、组织、菜单、会话等具体业务模型。
- 不要在本包导入 `internal`、`pkg` 或任何基础设施实现。

## 扩展规范

只有当字段确实是跨模块、跨层都需要理解的平台契约时，才可以扩展这里的类型。模块私有字段应留在对应模块的 `model`、`service` 或 handler DTO 中。
