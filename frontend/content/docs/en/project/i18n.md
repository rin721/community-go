# i18n

The default locale is `zh-CN`, and routing uses `no_prefix`. That means `/docs/components/actions` is the same path in all three languages; the active locale decides which collection is rendered.

## Locale Files

Shared user-facing copy lives in:

- `i18n/locales/zh-CN.json`
- `i18n/locales/en.json`
- `i18n/locales/ja.json`

When adding shared copy, update all three files. Some existing feature pages still contain inline Chinese; when touching those areas substantially, migrate reusable copy into locale files.

## Docs Content

Nuxt Content defines three collections: `docsZhCn`, `docsEn`, and `docsJa`. The directory structure and slugs stay identical across locales so language switches do not change URLs.

## Writing Guidance

Titles, descriptions, navigation labels, and prose should read naturally in the target language. Code identifiers, paths, and component names remain unchanged.
