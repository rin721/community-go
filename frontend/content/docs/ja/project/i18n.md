# i18n

デフォルト言語は `zh-CN` で、ルーティング戦略は `no_prefix` です。つまり `/docs/components/actions` は三言語で同じパスになり、現在の locale がどの collection を描画するかを決めます。

## Locale ファイル

共有のユーザー向け文言は次にあります。

- `i18n/locales/zh-CN.json`
- `i18n/locales/en.json`
- `i18n/locales/ja.json`

共有文言を追加するときは三つを同期します。一部の既存機能ページにはまだインライン中国語があるため、大きく触る場合は再利用文言を locale ファイルへ移します。

## Docs 内容

Nuxt Content は `docsZhCn`、`docsEn`、`docsJa` の三つの collection を定義します。言語を切り替えても URL が変わらないように、ディレクトリ構造と slug は同じにします。

## 書き方

タイトル、説明、ナビゲーション名、本文は対象言語で自然に書きます。コード識別子、ファイルパス、コンポーネント名はそのまま保ちます。
